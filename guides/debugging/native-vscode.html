Depuración: ARM nativo y VS Code

Depura un binario AArch64 en hardware ARM64 real con GDB local desde VS Code.

En ARM nativo no se usa QEMU. VS Code lanza GDB directamente sobre el binario AArch64 porque el procesador anfitrión ya puede ejecutarlo.

ImportanteUsa esta guía solo en ARM64 real

Verifica con uname -m. Si reporta x86_64, vuelve a Depuración: QEMU y VS Code.

Compila

cd 00-hello-minimo
cd src
make

Lanza GDB

make gdb

En VS Code usa la configuración Debug ARM64 (nativo).

Instala estas extensiones:

Extensión ID
C/C++ ms-vscode.cpptools
Makefile Tools ms-vscode.makefile-tools
ARM64 Assembly (GNU AS) Felipenguim.vscode-arm64-assembly
StackScope EightLayerStudio.StackScope

Config completa para copiar en .vscode/launch.json:

{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug ARM64 (nativo)",
      "type": "cppdbg",
      "request": "launch",
      "program": "${workspaceFolder}/src/build/main",
      "cwd": "${workspaceFolder}/src",
      "MIMode": "gdb",
      "miDebuggerPath": "/usr/bin/gdb",
      "stopAtEntry": true,
      "setupCommands": [
        { "text": "-enable-pretty-printing" },
        { "text": "set architecture aarch64" },
        { "text": "set disassemble-next-line on" },
        { "text": "set step-mode on" },
        { "text": "set pagination off" }
      ]
    }
  ]
}

Qué cambia frente a QEMU

Tema QEMU en x86_64 ARM nativo
Ejecución qemu-aarch64 Directa
Debugger gdb-multiarch gdb
Conexión localhost:1234 Launch local
Makefile activo Template QEMU copiado como Makefile Template ARM64 copiado como Makefile
TipMisma idea, menos capas

Los registros, syscalls y flujo del programa son los mismos. Lo que cambia es la forma en que el proceso llega a ejecutarse.

Errores visuales comunes

En ARM nativo también puede parecer que GDB salta líneas. Esto ocurre porque GDB puede avanzar por líneas fuente, mientras el procesador ejecuta instrucciones. Para assembly, usa comandos por instrucción en la Debug Console:

stepi
nexti
break _start
break printString
break readIntFromConsole
info registers x0 x1 x2 x8 pc
x/6i $pc

svc #0 entra al kernel. No esperes ver por dentro la syscall como si fuera una función de tu código.

TipBreakpoints por etiqueta

break nombreEtiqueta funciona si el símbolo existe en el binario. Con .include, la etiqueta queda en el mismo ensamblado; con .extern, debe venir de otro objeto enlazado.
