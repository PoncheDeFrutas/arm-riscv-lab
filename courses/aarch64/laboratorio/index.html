01 · Laboratorio ARM64 reproducible

Instala, ejecuta, inspecciona y depura el primer programa AArch64 sin perderte en detalles avanzados.

Página principal de la Unidad 01 de la ruta AArch64.

Cursos > AArch64 > Laboratorio ARM64 reproducible

Esta unidad prepara el laboratorio base. No estudia ELF, ABI, debugging avanzado ni syscalls en profundidad. El objetivo es que puedas instalar el entorno, elegir la ruta correcta, ejecutar un ejemplo mínimo y verificar que el binario AArch64 existe y corre.

No tengo Raspberry Pi

Primer ejemplo

Ver presentación

Mapa rápido

1. Elige ruta Raspberry Pi ARM64 real o x86_64 con QEMU user mode.

2. Instala Toolchain, binutils, Make, QEMU o GDB según tu máquina.

3. Ejecuta Entra a src, compila con make y corre con make run.

4. Observa Usa file, readelf, objdump, nm y strace como primera mirada.

5. Depura Detente en _start, mira registros y avanza instrucción por instrucción.

Qué ruta uso

Si tienes… Usa… Señal esperada
Raspberry Pi con Linux de 64 bits ARM64 nativo uname -m muestra aarch64 o arm64
Laptop o PC Linux x86_64 Cross toolchain + QEMU user mode uname -m puede mostrar x86_64
Máquina restringida o laboratorio compartido Docker opcional Mismo entorno dentro del contenedor
TipRecomendación docente

Si el estudiante no sabe cuál usar, debe empezar por x86_64 con QEMU user mode. Es la ruta más reproducible cuando no hay Raspberry Pi disponible.

Resultado esperado

Al terminar la unidad debes poder ejecutar:

cd 00-hello-minimo
cd src
make
make run

Salida esperada:

Hola ARM64

Y debes poder explicar, en una frase corta, qué herramienta genera el binario, qué herramienta lo ejecuta y qué herramienta permite mirarlo por dentro.
