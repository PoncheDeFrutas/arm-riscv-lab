15 · ABI, AAPCS64 e interoperabilidad con C

Calling convention formal para escribir assembly compatible con C, libc y el linker.

Página principal de la Unidad 15 de la ruta AArch64.

Cursos > AArch64 > ABI, AAPCS64 e interoperabilidad con C

Ya escribiste funciones, stack frames, syscalls y datos complejos. Ahora toca usar una regla compartida: qué espera una función antes de ser llamada, qué debe preservar y dónde deja el resultado.

Qué debe quedar claro

Una ABI no es una instrucción. Es un contrato entre piezas de código.

C
  llama funciones

assembly
  implementa funciones

ABI
  define cómo se pasan argumentos, retornos y responsabilidad de registros

Si C llama una función assembly, ambos lados deben estar de acuerdo. C no sabe qué registros elegiste por gusto. C espera AAPCS64.

Problema que resuelve

Sin calling convention formal, cada función tendría que explicar su propio formato de llamada:

función A
  argumento en x0
  resultado en x0

función B
  argumento en x5
  resultado en x2

función C
  destruye x19 sin avisar

Eso no escala. AAPCS64 evita ese caos. Define una base común para que código C, assembly, libc, objetos .o y librerías puedan trabajar juntos.

Mapa mental de una llamada

caller
  coloca argumentos en x0-x7
  guarda temporales si los necesita
  ejecuta bl

callee
  recibe argumentos
  preserva registros callee-saved si los usa
  deja resultado en x0
  ejecuta ret

La unidad no busca memorizar una tabla aislada. Busca que puedas leer una función y responder:

¿quién puso este valor aquí?
¿quién debe conservar este registro?
¿qué valor verá el caller al regresar?

Tabla mínima de registros generales

Registro Rol AAPCS64 inicial
x0-x7 argumentos y valores de retorno
x8 indirect result location / uso especial según contexto
x9-x15 temporales caller-saved
x16-x17 intra-procedure-call temporaries
x18 platform register, tratar con cuidado
x19-x28 callee-saved
x29 frame pointer
x30 link register
sp stack pointer, alineado a 16 bytes

Tabla mínima de floating/SIMD

Registro Rol inicial
v0-v7 argumentos y retornos floating/SIMD
v8-v15 preservación parcial según AAPCS64
v16-v31 temporales

Caller-saved y callee-saved

Concepto Responsable Idea
caller-saved caller si necesita el valor después de llamar, lo guarda
callee-saved callee si usa el registro, debe restaurarlo antes de retornar

Esta tabla es pequeña, pero cambia cómo escribes funciones. Antes podías usar registros como espacio temporal local. Ahora debes pensar quién confía en ellos.
